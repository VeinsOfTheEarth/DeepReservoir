total_timesteps (required for learn)
How many environment steps to run in total during training (summed over all episodes). 
This is the only thing you absolutely must set for SB3.

n_steps (optional; PPO-specific)
Number of steps per rollout per environment before doing a policy update (the “chunk size” PPO collects for advantage estimation). 
Larger values = less frequent but longer updates; SB3 default is 2048.

batch_size (optional)
Minibatch size used by the optimizer when training on each rollout; SB3 shuffles the n_steps * n_envs transitions 
and breaks them into batches of this size (default 64). You normally want batch_size <= n_steps * n_envs, often choosing a divisor of n_steps * n_envs.

n_epochs (optional)
How many passes PPO makes over the collected rollout per update (default 10). 
Higher → more optimization on the same data (can help sample efficiency, but too high can overfit/unstabilize).

n_episodes (not a direct hyperparam)
This is implied by total_timesteps and your environment’s episode length: roughly

n_episodes
≈
total_timesteps
/
episode_length
n_episodes≈total_timesteps/episode_length.
To “tune” it, you change either total_timesteps or how long your episodes are (via env logic like episode_length), not a separate argument to PPO.

Agent = PPO algorithm + neural nets + rollout buffer.
Environments = your DeepReservoir env(s), possibly vectorized (n_envs > 1).
Interaction loop = agent picks actions, env steps reservoir forward, returns rewards; agent stores transitions; periodically, PPO updates the nets.

2. Rollout + update timeline (what happens when)

Think along a time axis with nested loops:

for training_step in range(total_timesteps):
    1. Agent gets obs_t from each env
    2. Policy net πθ outputs action distribution for each obs_t
    3. Sample action_t ~ πθ(. | obs_t)
    4. Step envs with action_t → obs_{t+1}, reward_t, done_t, info_t
    5. Store (obs_t, action_t, reward_t, done_t, logπθ(action_t|obs_t), Vϕ(obs_t))
       in the rollout buffer

    When we've collected n_steps steps per env (i.e., n_steps * n_envs transitions):
        6. Compute returns & advantages (using gamma, gae_lambda)
        7. Train policy/value nets for n_epochs over this rollout
        8. Clear rollout buffer
